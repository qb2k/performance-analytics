# Performance Analytics — прототип ИС HMLPE

Веб-приложение для оценки эффективности участников ИТ-проекта  
на основе гибридной многоуровневой модели (HMLPE).

## Формула

```
Ei = α·Pi + β·Ri + γ·Vi + δ·Si,   где α+β+γ+δ = 1
```

| Слой | Назначение |
|------|-----------|
| P-layer | Процессная эффективность (скорость, стабильность) |
| R-layer | Ролево-функциональная эффективность |
| V-layer | Ценностный вклад (семантика задач по меткам) |
| S-layer | Социально-командный вклад (комментарии, реакции) |

## Запуск

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Запустить приложение
streamlit run app.py
```

Откроется браузер на http://localhost:8501

## Структура проекта

```
performance_analytics/
├── app.py              # Streamlit-приложение (UI)
├── hmlpe.py            # Вычислительное ядро (P/R/V/S layers)
├── config.yaml         # Веса и параметры модели
├── requirements.txt
├── README.md
└── data/
    ├── metrics.csv     # Метрики GitHub (issues, PR)
    └── users.csv       # Справочник участников (роли, уровни)
```

## Входные данные

### metrics.csv
Экспорт из GitHub Action `issue-metrics`. Поля:
- `id`, `type` (issue/pull_request), `author`, `labels`
- `created_at`, `closed_at`, `state`
- `time_to_close_hours`, `time_to_first_response_hours`, `comments`

### users.csv
| Поле | Описание |
|------|---------|
| login | GitHub-логин |
| role | backend / frontend / qa / pm / data scientist |
| level | junior / middle / senior |
| team | название команды |

### config.yaml
Задаёт веса слоёв модели, ролевые коэффициенты и маппинг меток на ценность.

## Функционал

- ✅ Загрузка демо-данных или пользовательских файлов
- ✅ Интерактивная настройка весов α, β, γ, δ через ползунки
- ✅ Рейтинговая таблица с цветовой индикацией
- ✅ Радарные диаграммы слоёв
- ✅ Тепловая карта участников
- ✅ Сравнение команд
- ✅ Профиль каждого участника
- ✅ Экспорт в Excel (с графиком) и CSV
- ✅ Воспроизводимость результатов
